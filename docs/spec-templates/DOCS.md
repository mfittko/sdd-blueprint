# Documentation Spec Template

**Issue Title:** `docs: <area to clarify>`

## Summary
Brief description of the documentation gap or update required.

## Target Audiences
- Primary readers (e.g., contributors, operators, stakeholders)
- Secondary readers, if any

## Source Material
- Existing docs that must be updated
- External references (design docs, diagrams, decision records)

## Deliverables
- [ ] Doc/page 1 updated or created
- [ ] Doc/page 2 updated or created
- [ ] Screenshots/diagrams refreshed (if applicable)

## Voice & Style Notes
- Tone (instructional, reference, tutorial)
- Required sections (e.g., prerequisites, examples, FAQs)

## Review Checklist
- Technical reviewer assigned
- Proofreading complete
- Accessibility reviewed (alt text, headings, etc.)

## Verification
- Commands or steps the reviewer should run to validate accuracy
- Links to preview environments or published docs
